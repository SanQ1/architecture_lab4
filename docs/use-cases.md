1. **Назва**: Створення замовлення  
    **Актор**: Клієнт  
    **Передумови**: Користувач автентифікований, замовлення не пусте.  
    **Основний сценарій**:  
-       Користувач натискає "Створити замовлення".  
-       Система перевіряє чи не є замовлення пустим.  
-       Система створює замовлення.  
    **Альтернативні сценарії/помилки**:  
-       Замовлення пусте: помилка.  
-       Користувача не існує: помилка 409.  

2. **Назва**: Видалення замовлення  
    **Актор**: Клієнт  
    **Передумови**: Користувач автентифікований, замовлення існує.  
    **Основний сценарій**:  
-       Користувач натискає "Видалити замовлення".  
-       Система перевіряє чи існує замовлення.  
-       Система прибирає замовлення.  
    **Альтернативні сценарії/помилки**:  
-       Замовлення не існує: помилка 404.  
-       Користувача не існує: помилка 409.  

3. **Назва**: Додати товар  
    **Актор**: Адміністратор  
    **Передумови**: Адміністратор автентифікований.  
    **Основний сценарій**:  
-       Адміністратор вводить необхідну інформацію про товар.  
-       Натискає "Додати".
-       Система додає товар.  
    **Альтернативні сценарії/помилки**:  
-       Адміністратор не існує: помилка 409.  
  
4. **Назва**: Прибрати товар  
    **Актор**: Адміністратор  
    **Передумови**: Товар існує, адміністратор автентифікований.  
    **Основний сценарій**:  
-       Адміністратор обирає товар.  
-       Натискає "Видалити".  
-       Система прибирає відповідний товар.  
    **Альтернативні сценарії/помилки**:  
-       Адміністратора не існує: помилка.  
-       Товар не існує: помилка 404.  
  
5. **Назва**: Перегляд товарів  
    **Актор**: Клієнт  
    **Передумови**: Клієнт автентифікований.  
    **Основний сценарій**:  
-       Клієнт натискає "Перегляд товарів".  
    **Альтернативні сценарії/помилки**:  
-       Користувача не існує: помилка 409.  
  
  
6. **Назва**: Реєстрація  
    **Актор**: Клієнт/Адміністратор  
    **Основний сценарій**:  
-       Користувач вводить логін, пароль.  
-       Система перевіряє унікальність логіну.  
-       Користувач створюється.  
    **Альтернативні сценарії/помилки**:  
-       Користувач вже існує: помилка.  
  
7. **Назва**: Авторизація  
    **Актор**: Клієнт/Адміністратор  
    **Основний сценарій**:  
-       Користувач вводить логін, пароль.  
-       Система перевіряє наявість користувача, та правильність паролю.  
-       Користувач входить в свій профіль.  
    **Альтернативні сценарії/помилки**:  
-       Користувача не існує: помилка.  
-       Пароль не правильний: вхід не здійснюється.
