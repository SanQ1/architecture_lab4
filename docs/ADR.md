**Підхід**: Anemic Domain Model
**Обґрунтування**: Використання Anemic Domain Model дозволяє чітко розділити стан та поведінку системи, залишаючи класи моделей чистими, легкими структурами даних. Це унеможливлює змішування бізнес-правил із логікою інфраструктури чи ORM та переносить усю координацію транзакцій, генерацію ідентифікаторів та сценарії використання в окремий сервісний шар. Такий підхід робить архітектуру гнучкішою, спрощує мапінг об'єктів.
